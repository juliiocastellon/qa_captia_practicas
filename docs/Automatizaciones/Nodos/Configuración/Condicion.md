# 🧩 Despiece de Elementos Funcionales

| # | Nombre del Elemento | Tipo | Destino / Acción | Descripción Funcional |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Nodo "Condición" | Nodo lógico | Configura lógica condicional | Permite visualizar y editar un nodo encargado de bifurcar el flujo del workflow según reglas configuradas por el usuario. |
| 1.1 | Sección "Configuración básica" | Configuración general | Configura información básica | Permite definir los datos básicos identificativos del nodo. |
| 1.1.1 | Campo "Nombre del nodo" | Campo de texto | Renombra el nodo | Permite asignar un nombre identificativo al nodo dentro del workflow. Para utilizarlo, el usuario debe escribir el nombre que quiera mostrar dentro de la automatización. |
| 1.1.2 | Campo "Descripción" | Campo de texto | Añade descripción | Permite añadir una explicación o contexto sobre el funcionamiento del nodo. Para utilizarlo, el usuario debe escribir la descripción deseada dentro del campo correspondiente. |
| 1.2 | Sección "Configuración específica" | Configuración lógica | Configura placeholders y reglas | Permite definir las variables y condiciones que utilizará el nodo para tomar decisiones dentro del workflow. |
| 1.2.1 | Configuración "Placeholders" | Configuración de entradas | Añade variables de entrada | Permite definir placeholders o variables que serán utilizadas posteriormente dentro de las reglas del nodo. Para utilizarlo, el usuario debe añadir placeholders y seleccionar el tipo correspondiente. |
| 1.2.2 | Configuración "Reglas" | Configuración lógica | Define condiciones | Permite crear reglas condicionales para determinar cómo se bifurcará el flujo del workflow. Para utilizarlo, el usuario debe seleccionar variables, operadores y valores para construir la condición deseada. |
| 1.3 | Acciones del nodo | Acciones principales | Guarda o cancela cambios | Permite guardar la configuración realizada o cancelar los cambios efectuados sobre el nodo. Para utilizarlo, el usuario debe pulsar “Guardar cambios” o “Cancelar” según la acción deseada. |

# 💡 Guía de Uso

El nodo **Condición** está diseñado para controlar bifurcaciones y decisiones dentro de automatizaciones y workflows.

Se utiliza principalmente para:

- Crear lógica condicional
- Controlar flujos automáticos
- Aplicar reglas
- Comparar valores
- Gestionar decisiones automáticas
- Construir automatizaciones
- Organizar comportamientos lógicos

## ✅ Cómo configurar correctamente el nodo Condición

1. Arrastra el nodo al canvas del workflow.
2. Configura el nombre y descripción del nodo.
3. Añade los placeholders necesarios.
4. Define las reglas y condiciones que se evaluarán.
5. Guarda los cambios realizados.
6. Conecta el nodo con el resto del workflow.
