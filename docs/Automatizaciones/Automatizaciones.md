# 🖥️ Automatizaciones

**Objetivo principal:** Permite al usuario visualizar, crear y gestionar workflows automáticos para ejecutar acciones, reglas y procesos industriales de forma automatizada dentro de la plataforma.

## 🧩 Despiece de elementos funcionales

| # | Nombre del elemento | Tipo | Destino / Acción | Descripción funcional |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Contador de workflows | Indicador informativo | — | Permite al usuario visualizar el número total de workflows disponibles y comprobar cuántos se encuentran activos, inactivos o disponibles dentro del entorno actual. |
| 2 | Buscador de workflows | Campo de búsqueda | Filtra workflows | Permite al usuario localizar workflows rápidamente escribiendo su nombre o parte del mismo. Para utilizarlo, el usuario debe hacer clic sobre el buscador e introducir el texto deseado para filtrar automáticamente el listado. |
| 3 | Filtro de estado | Filtro desplegable | Filtra automatizaciones | Permite al usuario visualizar workflows según su estado actual, como activo, borrador o fallido. Para utilizarlo, el usuario debe abrir el desplegable y seleccionar el estado que desea mostrar. |
| 4 | Filtro de tipo | Filtro desplegable | Filtra por tipo | Permite al usuario filtrar workflows según el tipo de automatización o trigger configurado. Para utilizarlo, el usuario debe seleccionar el tipo deseado desde el menú desplegable disponible. |
| 5 | Toggle workflows de test | Toggle/Filtro | Mostrar workflows de test | Permite al usuario mostrar u ocultar workflows utilizados para pruebas o testing. Para utilizarlo, el usuario debe activar o desactivar el toggle correspondiente según desee visualizar estos workflows. |
| 6 | Botón de logs de ejecución | CTA Secundario | Ver ejecuciones | Permite al usuario acceder al historial y registros de ejecución de workflows y automatizaciones. Para utilizarlo, el usuario debe pulsar el botón de logs y consultar la información de cada ejecución realizada. |
| 7 | Selección múltiple | Herramienta de selección | Seleccionar workflows | Permite al usuario seleccionar varios workflows al mismo tiempo para realizar acciones conjuntas. Para utilizarlo, el usuario debe activar el modo de selección múltiple y marcar los workflows deseados. |
| 8 | Botón "Crear workflow" | CTA Principal | Crear automatización | Permite al usuario iniciar la creación de un nuevo workflow o automatización. Para utilizarlo, el usuario debe pulsar el botón, configurar los triggers, acciones y condiciones necesarias y guardar la automatización creada. |
| 9 | Listado de workflows | Grid de tarjetas | Accede a workflow | Permite al usuario visualizar todos los workflows disponibles mediante tarjetas informativas. Para acceder a uno concreto, el usuario debe hacer clic sobre la tarjeta correspondiente para visualizarlo, editarlo o gestionarlo según sus permisos. |

## 💡 Guía de uso

La sección de Automatizaciones centraliza todos los workflows y reglas automáticas configuradas dentro de la plataforma. Desde esta vista puedes buscar automatizaciones concretas, filtrarlas por estado o tipo y consultar rápidamente el estado de ejecución de cada una.

Cada tarjeta representa un workflow independiente e incluye información relevante como triggers configurados, fecha de última ejecución y estado operativo. Desde esta sección también es posible consultar logs de ejecución, seleccionar múltiples workflows para gestión masiva y crear nuevas automatizaciones adaptadas a distintos procesos industriales.
